# Skills 一覧

このプロジェクトで利用可能な Claude Code スキルの一覧です。

## スキル一覧

| スキル名 | コマンド例 | 概要 |
|----------|-----------|------|
| [add-doc](#add-doc) | `/add-doc packages/@ai-suburi/core/chapter3/test3-9-duckduckgo-search.ts` | ソースコードからドキュメントセクションを自動生成・追記 |
| [review-doc](#review-doc) | `/review-doc chapter3` | ドキュメントの正確性・整合性チェック＆修正 |
| [brushup-doc](#brushup-doc) | `/brushup-doc chapter3` | ドキュメントの文章品質向上＆内容充実化 |
| [cleanup-code](#cleanup-code) | `/cleanup-code packages/@ai-suburi/core/chapter3/test3-12.ts` | TypeScriptコードの型エラー修正・非推奨API置換・未使用import削除・JSDoc追加 |

## 推奨ワークフロー

```
1. ソースコードを新規作成
       ↓
  (optional) /cleanup-code で型エラー修正・import整理・JSDoc追加
       ↓
2. /add-doc でドキュメントセクションを自動生成
       ↓
  (optional) /review-doc でコードとの整合性をチェック
       ↓
3. /brushup-doc で文章品質・内容を仕上げ
```

## add-doc

**ソースコードからドキュメントを自動生成して追記するスキル**

- **引数**: ソースコードのファイルパス
- **使用ツール**: Read, Edit, Glob, Grep

### やること

1. 指定されたソースコードを読み込み、内容を把握
2. 対応する章の `index.md` を特定
3. 既存セクションのスタイルに合わせて新しいセクションを追記

### 追記される内容

- セクション見出し・概要説明
- サンプルコードの説明（箇条書き）
- ソースコード全文（コードブロック）
- 実行方法
- 概要テーブル・「この章で学ぶこと」への項目追加

## review-doc

**ドキュメントの正確性と構成の整合性をチェック・修正するスキル**

- **引数**: 章番号 or ドキュメントパス（例: `chapter3`, `3`）
- **使用ツール**: Read, Edit, Glob, Grep

### やること

1. ドキュメント内のコードブロックと実際のソースコードを行単位で差分比較
2. 概要テーブル・「この章で学ぶこと」・前提条件の整合性チェック
3. 見出しの重複チェック（MD024対策）
4. 未ドキュメントのソースファイル検出
5. 実行コマンド・ファイル名の正確性チェック

### 出力

修正項目と問題なし項目のチェックリストをサマリーとして出力する。

## brushup-doc

**ドキュメントの文章品質を向上させ、内容を充実化するスキル**

- **引数**: 章番号 or ドキュメントパス（例: `chapter3`, `3`）
- **使用ツール**: Read, Edit, Glob, Grep, WebSearch

### やること

#### 概念・トピックの網羅性チェック

- テーブルに名前だけ挙がっていて詳細セクションがない概念の検出・追加
- 各セクションに「なぜ必要か（Why）」の説明があるか確認
- 関連する概念同士のつながりの明示（例: CoT→ReAct の Thought 部分）

#### 図解・ビジュアルの改善

- ASCII アート図の mermaid 記法への変換（フローチャート、サイクル図など）
- プロセスやサイクルの説明に mermaid 図を新規追加
- 変換時のラベル・注釈の欠落チェック

#### 文章品質の改善

- 専門用語の初出に補足追加
- 抽象的な説明の具体化・身近なシナリオ例の追加
- 長文の分割・冗長表現の除去
- 接続詞の改善・文体の統一

#### 内容の充実化

- 「なぜこの機能が必要か」の概要説明強化
- コード解説の補足（パラメータの意味、選択理由など）
- バリエーションごとの実装例追加（例: Zero-shot / Few-shot）
- 類似概念の比較テーブル追加（違い・使い分けの判断基準）
- Tips・注意事項のアドモニション追加（`:::tip`, `:::caution`, `:::info`）
- 実行結果の例示追加
- 参考文献リンクの拡充

### 出力

概念の網羅性・図解の改善・文章品質の改善・内容の充実化・参考文献の追加をカテゴリ別にサマリーとして出力する。

## cleanup-code

TypeScript コードの型エラー修正・未使用 import 削除・JSDoc 追加を行うスキル。

- **引数**: TypeScript ファイルのパス
- **使用ツール**: Read, Edit, Glob, Grep, Bash

### 整理内容

1. `tsc --noEmit` で型エラーを検出・修正
2. `@deprecated` な API を検出・代替 API に置換
3. 未使用の import を検出・削除
4. JSDoc がない関数に JSDoc を追加

### 整理結果の出力

型エラー修正・未使用 import 削除・JSDoc 追加の結果をカテゴリ別にサマリーとして出力する。
