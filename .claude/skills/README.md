# Skills 一覧

このプロジェクトで利用可能な Claude Code スキルの一覧です。

## スキル一覧

| スキル名 | コマンド例 | 概要 |
|----------|-----------|------|
| [add-doc](#add-doc) | `/add-doc packages/@ai-suburi/core/chapter3/test3-9-duckduckgo-search.ts` | ソースコードからドキュメントセクションを自動生成・追記 |
| [review-doc](#review-doc) | `/review-doc chapter3` | ドキュメントの正確性・整合性チェック＆修正 |
| [brushup-doc](#brushup-doc) | `/brushup-doc chapter3` | ドキュメントの文章品質向上＆内容充実化 |

## 推奨ワークフロー

```
1. ソースコードを新規作成
       ↓
2. /add-doc でドキュメントセクションを自動生成
       ↓
3. /review-doc でコードとの整合性をチェック
       ↓
4. /brushup-doc で文章品質・内容を仕上げ
```

---

## add-doc

**ソースコードからドキュメントを自動生成して追記するスキル**

- **引数**: ソースコードのファイルパス
- **使用ツール**: Read, Edit, Glob, Grep

### やること

1. 指定されたソースコードを読み込み、内容を把握
2. 対応する章の `index.md` を特定
3. 既存セクションのスタイルに合わせて新しいセクションを追記

### 追記される内容

- セクション見出し・概要説明
- サンプルコードの説明（箇条書き）
- ソースコード全文（コードブロック）
- 実行方法
- 概要テーブル・「この章で学ぶこと」への項目追加

---

## review-doc

**ドキュメントの正確性と構成の整合性をチェック・修正するスキル**

- **引数**: 章番号 or ドキュメントパス（例: `chapter3`, `3`）
- **使用ツール**: Read, Edit, Glob, Grep

### やること

1. ドキュメント内のコードブロックと実際のソースコードを行単位で差分比較
2. 概要テーブル・「この章で学ぶこと」・前提条件の整合性チェック
3. 見出しの重複チェック（MD024対策）
4. 未ドキュメントのソースファイル検出
5. 実行コマンド・ファイル名の正確性チェック

### 出力

修正項目と問題なし項目のチェックリストをサマリーとして出力する。

---

## brushup-doc

**ドキュメントの文章品質を向上させ、内容を充実化するスキル**

- **引数**: 章番号 or ドキュメントパス（例: `chapter3`, `3`）
- **使用ツール**: Read, Edit, Glob, Grep, WebSearch

### やること

#### 文章品質の改善

- 専門用語の初出に補足追加
- 抽象的な説明の具体化
- 長文の分割・冗長表現の除去
- 接続詞の改善・文体の統一

#### 内容の充実化

- 「なぜこの機能が必要か」の概要説明強化
- コード解説の補足（パラメータの意味、選択理由など）
- Tips・注意事項のアドモニション追加（`:::tip`, `:::caution`, `:::info`）
- 類似機能の比較テーブル追加
- 実行結果の例示追加
- 参考文献リンクの拡充

### 出力

文章品質の改善・内容の充実化・参考文献の追加をカテゴリ別にサマリーとして出力する。
